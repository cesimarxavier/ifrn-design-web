Design de Interfaces

UI/UX e Processo de Design Centrado no Ser Humano

Nesta disciplina você vai dominar o Figma como ferramenta profissional de design, aprender a criar interfaces visuais consistentes e acessíveis, e mergulhar no processo completo de design centrado no usuário: da pesquisa ao protótipo testado e refinado.

Ao final da disciplina você será capaz de

Domínio do Figma

Criar interfaces profissionais com componentes, estilos, auto-layout e protótipos navegáveis

Visual Design Consistente

Aplicar hierarquia visual, tipografia, cor e espaçamento com acessibilidade WCAG

Processo de Design

Conduzir pesquisa com usuários, criar personas, mapear jornadas e testar soluções

Projeto Completo

Desenvolver projeto de interface do problema à solução testada e documentada

Sobre a Disciplina

Para quem é

Esta disciplina é ideal para estudantes que já possuem conhecimentos básicos de design visual e querem mergulhar no mundo do design de interfaces digitais. Você vai aprender tanto a parte técnica (Figma profissional) quanto a metodologia de design centrado no usuário, preparando-se para criar produtos digitais reais.

Duração

40 horas distribuídas em 2 bimestres (20 aulas cada)

Estrutura

Bimestre 1: Visual Design / Bimestre 2: Processo de Design

Objetivo

Dominar ferramentas e processos profissionais de UI/UX Design

Dois bimestres complementares

  • Bimestre 1: Visual Design de Interfaces no Figma (20 aulas)

    Domínio completo do Figma: hierarquia visual, tipografia, cor, espaçamento, componentes, estados, responsividade e prototipação. Objetivo: criar interfaces limpas, coerentes e acessíveis.
  • Bimestre 2: Processo de Design e Projeto (20 aulas)

    Design centrado no ser humano: pesquisa, personas, jornadas, arquitetura de informação, prototipação, testes de usabilidade e redesign iterativo. Objetivo: projeto completo documentado.

Como você aprende aqui?

A disciplina combina aulas teóricas curtas com prática intensiva em laboratório. Cada aula tem uma tarefa para casa que consolida o aprendizado.

Teoria + Exemplos

Exposição teórica focada nos conceitos essenciais, com análise de exemplos reais de interfaces e demonstrações práticas no Figma.

  • • Conceitos fundamentais de UI/UX
  • • Demonstrações ao vivo no Figma
  • • Análise de casos reais

Prática Intensiva

Trabalho hands-on no Figma com exercícios progressivos, feedback individualizado e construção incremental de projetos reais.

  • • Exercícios práticos individuais/grupo
  • • Acompanhamento personalizado
  • • Tarefas para casa que consolidam

Progressão Incremental

A disciplina segue uma progressão cuidadosa: do simples ao complexo, do guiado ao autônomo, sempre construindo sobre o que foi aprendido anteriormente.

1

Bimestre 1: Visual Design de Interfaces no Figma

Aulas 1-20 · Fundamentos + UI Kit + Protótipo

Objetivo do Bimestre

Dar repertório visual + domínio básico de Figma para construir interfaces limpas, coerentes e minimamente acessíveis, preparando terreno para o projeto do 2º bimestre.

Habilidades e competências

Figma completo, hierarquia visual, tipografia, cor e contraste WCAG, espaçamento, componentes, variantes, estados, responsividade, prototipação

Entregável do Bimestre

Arquivo Figma organizado com UI Kit v1 + pelo menos 2 telas (desktop + mobile) + protótipo navegável simples

Cronograma Aula a Aula

Aula 1
Kick-off
Kick-off + visão geral

Em sala: Apresentar disciplina, projeto anual, diferenças UI/UX. Criar conta Figma, tour de interface. Ver exemplos de boas e péssimas interfaces.

Tarefa: Levantar 3 interfaces (sites/apps) que gostam e 3 que odeiam, com print e 1 frase justificando cada.

Aula 2
Fundamentos
Hierarquia visual + frames

Em sala: Conceitos de hierarquia, alinhamento, proximidade. No Figma: criar frames, grid simples e replicar layout estático fornecido.

Tarefa: Finalizar a tela-referência se não concluir em aula.

Aula 3
Tipografia
Tipografia em interfaces

Em sala: Legibilidade x leiturabilidade, hierarquia tipográfica, uso de poucas fontes. No Figma: criar estilos de texto (títulos, subtítulo, corpo, legenda).

Tarefa: Aplicar os estilos de texto em uma tela simples (ex.: página de notícia) usando as fontes indicadas.

Aula 4
Cor
Cor e contraste

Em sala: Teoria básica de cor para UI, contraste e estados. No Figma: criar estilos de cor (primária, secundária, neutros, feedback: sucesso/erro/aviso). Testar contraste com plugin.

Tarefa: Ajustar paleta para garantir contraste mínimo em pelo menos 2 combinações (texto x fundo). Printar antes/depois.

Aula 5
Espaçamento
Espaçamento e grid

Em sala: Conceito de ritmo visual, grid 8pt. Introdução ao Auto Layout para alinhamento e espaçamento.

Tarefa: Refazer um pequeno card usando Auto Layout, mantendo espaçamentos coerentes.

Aula 6
Componentes
Componentes básicos

Em sala: Definir o que é componente. No Figma: criar botões (primário/secundário), campos de formulário e cards simples como componentes.

Tarefa: Criar uma pequena "prateleira" com 3 variações de card usando os componentes.

Aula 7
Estados
Estados de interface

Em sala: Falar de estados (default, hover, pressed, disabled, erro). No Figma: usar variantes para criar estados de um botão e de um campo.

Tarefa: Montar uma mini "tabela de estados": 1 botão + 1 campo com todos os estados organizados.

Aula 8
Wireframe
Wireframe de baixa fidelidade

Em sala: Conceito de low-fi: foco na estrutura, não na beleza. No Figma: criar wireframe cinza (sem cor) de uma tela principal (ex.: home de app simples).

Tarefa: Completar o wireframe, garantindo hierarquia de informação (título, seções, CTA claro).

Aula 9
Hi-Fi
Layout visual de uma tela

Em sala: Transformar o wireframe da aula anterior em tela de alta fidelidade, usando tipografia e cores definidos.

Tarefa: Refinar a tela, ajustar alinhamentos, espaçamentos e textos.

Aula 10
Review
Crítica guiada (design review)

Em sala: Apresentações rápidas das telas em duplas/grupos. Discussão guiada sobre hierarquia, contraste, alinhamento. Apontar padrões de erro.

Tarefa: Revisar a própria tela com base no feedback, registrar 3 melhorias realizadas.

Aula 11
Responsivo
Responsividade – visão geral

Em sala: Conceitos básicos de design responsivo. No Figma: mostrar constraints, layout grid e frames de diferentes tamanhos (mobile/desktop).

Tarefa: Escolher a tela principal e esboçar a versão mobile (low-fi) em um frame separado.

Aula 12
Responsivo
Responsividade – prática

Em sala: Transformar o low-fi mobile em layout visual. Ajustar tipografia, espaçamentos e hierarquia para o formato menor.

Tarefa: Revisar ambas as versões (desktop e mobile) e listar 3 adaptações importantes entre elas.

Aula 13
Padrões
Listas, cards e leitura

Em sala: Padrões de listas, cards e "scanability". No Figma: criar uma tela de listagem (ex.: lista de cursos, produtos ou contatos) usando cards repetidos.

Tarefa: Completar a tela de listagem e garantir consistência entre os cards (mesmas medidas, alinhamentos).

Aula 14
Formulários
Formulários

Em sala: Boas práticas de formulários: label, placeholder, mensagens de erro, ajuda. No Figma: desenhar um formulário simples (ex.: cadastro básico).

Tarefa: Escrever mensagens de erro claras e humanas para 3 campos (nome, e-mail, senha) direto na interface.

Aula 15
UI Kit
Mini UI kit (v1)

Em sala: Consolidar tudo: cores, tipografia, botões, campos, cards, mensagens. No Figma: montar uma página "UI Kit – v1" com todos os elementos organizados.

Tarefa: Completar o UI kit e nomear os estilos e componentes de forma padronizada.

Aula 16
Organização
Organização do arquivo

Em sala: Naming, páginas (cover, UI kit, screens), seções, uso de grids consistentes. No Figma: reorganizar o arquivo da turma conforme um padrão definido.

Tarefa: Fazer print da estrutura de páginas e layers para entregar como registro de organização.

Aula 17
A11y
Acessibilidade visual

Em sala: Retomar contraste, tamanho mínimo de fontes e foco. Mostrar exemplos bons e ruins. Ajustar 1 tela para ficar mais acessível.

Tarefa: Entregar antes/depois de uma tela onde melhorou acessibilidade (fonte, contraste, espaçamento).

Aula 18
Studio
Revisão orientada + ajustes

Em sala: Acompanhamento em estúdio: circular, fazer micro-reviews, ajustar detalhes. Foco em deixar 2 telas "apresentáveis" (desktop + mobile).

Tarefa: Finalizar telas e garantir que tudo esteja usando componentes e estilos, não elementos "soltos".

Aula 19
Protótipo
Protótipo navegável

Em sala: Introdução à prototipação no Figma (links, hotspots, transições simples). Criar fluxo básico entre 3–4 telas (ex.: home → lista → detalhes → formulário).

Tarefa: Testar o próprio protótipo em casa/celular e anotar 3 pontos que confundem ou podem melhorar.

Aula 20
Entrega
🎯 Apresentação + fechamento do bimestre

Em sala: Pitches rápidos (2–3 min) por grupo/aluno, mostrando UI kit + protótipo. Feedback estruturado: pontos fortes, melhorias, aderência ao processo.

Tarefa: Autoavaliação escrita (o que aprendeu, onde ainda está travado). Sem tarefa pesada.

2

Bimestre 2: Processo de Design e Projeto

Aulas 21-40 · Design Centrado no Ser Humano

Objetivo do Bimestre

Sair do "layout bonito" e entrar em processo de design centrado no ser humano: problema → pesquisa → síntese → solução → teste → redesign, culminando em um mini projeto documentado.

Habilidades e competências

Briefing, pesquisa qualitativa, entrevistas, personas, jornadas do usuário, task flows, arquitetura de informação, wireframes, testes de usabilidade, iteração, documentação de processo

Entregável do Bimestre

Um mini projeto completo: Problema + contexto + personas/jornada + Fluxos principais + mapa de telas + Protótipo navegável em Figma + Ajustes pós-teste + Documentação enxuta (1 slide deck / one-pager)

Cronograma Aula a Aula

Aula 21
Kick-off
Visão de processo + briefing

Em sala: Apresentar projeto do bimestre. Explicar design centrado no ser humano e ciclo iterativo. Em grupo, escolher tema/problema (ex.: app simples para contexto escolar/local).

Tarefa: Refinar o problema em 1 parágrafo e listar 3 hipóteses sobre o usuário.

Aula 22
Contexto
Contexto e stakeholders

Em sala: Mapear contexto, stakeholders e objetivos de negócio/serviço. Construir um mini canvas (problema, público, objetivos, restrições).

Tarefa: Entrevistar 1 pessoa (ou simulação guiada) sobre o tema, anotando respostas-chave.

Aula 23
Pesquisa
Métodos de coleta (entrevista/questionário)

Em sala: Explicar entrevistas e questionários simples. Em sala: preparar roteiro de 5–7 perguntas abertas para entender necessidades.

Tarefa: Aplicar o roteiro com pelo menos 2 pessoas (ou usar dados fornecidos, se não for viável).

Aula 24
Síntese
Análise e síntese

Em sala: Trazer dados coletados. Em sala: agrupar respostas em padrões (post-its ou Figma FigJam), identificar dores, necessidades e oportunidades.

Tarefa: Registrar 3 insights principais e 3 oportunidades de solução.

Aula 25
Personas
Personas simples

Em sala: Explicar persona (não estereótipo, mas síntese). Em sala: criar 1–2 personas básicas com base nos dados.

Tarefa: Ajustar personas e escrever um pequeno texto: "Um dia na vida de [persona] usando/sem usar a solução".

Aula 26
Jornada
Jornada do usuário

Em sala: Explicar jornada (etapas, ações, dores, emoções). Em sala: construir jornada da persona para um cenário específico.

Tarefa: Completar jornada e destacar em outra cor os pontos de dor prioritários.

Aula 27
Fluxos
Task flows + requisitos

Em sala: Explicar task flow e requisitos funcionais. Em sala: definir 1–2 fluxos principais (ex.: cadastrar, buscar, enviar algo) e listar requisitos mínimos.

Tarefa: Produzir versão limpa dos fluxos em Figma/diagramador, com início, passos e resultado.

Aula 28
IA
Arquitetura de informação

Em sala: Apresentar sitemap/fluxo de telas. Em sala: desenhar mapa de telas mínimo da solução (sem detalhar UI).

Tarefa: Ajustar mapa de telas e validar se cobre os fluxos principais definidos.

Aula 29
Wireframe
Wireframes de baixa fidelidade

Em sala: Voltar ao Figma: criar wireframes low-fi das principais telas (sem polir visual). Foco em estrutura e priorização de conteúdo.

Tarefa: Completar wireframes das telas principais, garantindo coerência entre elas.

Aula 30
Hi-Fi
Do wireframe ao visual (reuso do UI kit)

Em sala: Conectar com 1º bimestre: aplicar UI kit v1 aos wireframes. Ajustar se necessário (UI kit pode evoluir).

Tarefa: Refinar ao menos 3 telas-chave no visual final.

Aula 31
Protótipo
Protótipo navegável (v2)

Em sala: Ampliar protótipo no Figma para cobrir o fluxo principal definido (ex.: cadastro completo, busca + detalhe).

Tarefa: Testar o protótipo com alguém em casa/colega e registrar dificuldades observadas.

Aula 32
Testes
Preparando testes de usabilidade

Em sala: Explicar teste de usabilidade simples: tarefas, observação, anotações. Em sala: cada grupo define 3–4 tarefas para os usuários testarem o protótipo.

Tarefa: Preparar folha de anotações (em papel ou digital) com tarefas e campos para observações.

Aula 33
Testes
Testes em sala (entre pares)

Em sala: Rodar testes em duplas/grupos: um é "usuário", outro observa. Registrar erros, dúvidas, cliques perdidos.

Tarefa: Consolidar achados em 5–7 problemas mais relevantes, com exemplos.

Aula 34
Priorizar
Priorização de problemas

Em sala: Introduzir matriz impacto x esforço. Em sala: priorizar os problemas encontrados e decidir o que será corrigido no redesign.

Tarefa: Escolher 3 problemas prioritários e escrever qual mudança de interface pretende fazer para cada.

Aula 35
Redesign
Redesign

Em sala: Ajustar telas no Figma com base nos problemas priorizados. Comparar versão antes/depois.

Tarefa: Finalizar redesign das telas escolhidas e atualizar o protótipo navegável.

Aula 36
Microcopy
Microcopy e feedbacks na interface

Em sala: Focar em texto da interface (labels, mensagens, estados vazios, erros). Em sala: revisar textos nas telas principais para deixá-los claros e consistentes.

Tarefa: Revisar todos os textos da solução e registrar 3 decisões de microcopy que melhoraram a experiência.

Aula 37
Docs
Documentação enxuta do projeto

Em sala: Estruturar um "dossiê compacto": problema, público, principais descobertas, solução proposta, fluxo principal, telas-chave, aprendizados dos testes.

Tarefa: Completar a documentação (slide, one-pager ou página no Figma) para apresentação final.

Aula 38
Pitch
Preparação do pitch

Em sala: Orientar storytelling: antes → problema → processo → solução → evidências (testes) → próximos passos. Ensaiar pitches em grupo.

Tarefa: Ajustar a apresentação/pitch com base no feedback recebido.

Aula 39
Apresentar
Apresentações finais

Em sala: Cada grupo apresenta o projeto (tempo cronometrado). Feedback estruturado: pontos fortes, melhorias, aderência ao processo.

Tarefa: Entregar versão final do arquivo Figma + documentação.

Aula 40
Entrega Final
🎯 Fechamento e reflexão

Em sala: Rodar uma retro: o que funcionou, o que não funcionou, o que aprenderam de UI, UX e processo. Conectar com demais disciplinas (programação).

Tarefa: Autoavaliação escrita: onde se enxergam mais fortes (visual, processo, pesquisa, protótipo etc.).

Avaliação e Projetos

Método de Avaliação

A disciplina adota avaliação formativa e somativa ao longo dos 2 bimestres, com foco na evolução do processo além do produto final.

O objetivo é avaliar tanto o domínio técnico (Figma, visual design) quanto a capacidade de conduzir processo (pesquisa, teste, iteração, documentação).

Componentes da Nota

  • 20% Participação, assiduidade e tarefas para casa
  • 30% Exercícios práticos em sala (progresso semanal)
  • 25% Entregável Bimestre 1: UI Kit + Protótipo navegável
  • 25% Entregável Bimestre 2: Projeto completo documentado

Critérios de Qualidade

  • ✓ Técnica: Uso correto de componentes, estilos, Auto Layout
  • ✓ Visual: Hierarquia clara, contraste acessível, consistência
  • ✓ Processo: Evidência de pesquisa, testes, iteração
  • ✓ Documentação: Clareza na comunicação das decisões
  • ✓ Organização: Estrutura de arquivo limpa e profissional

Entregáveis por Bimestre

Cada bimestre culmina em um entregável que demonstra domínio dos conceitos:

B1: UI Kit + Protótipo Navegável

Arquivo Figma organizado com UI Kit v1 + 2 telas (desktop + mobile) + protótipo com links funcionais entre 3-4 telas

B2: Projeto Completo Documentado

Problema + Pesquisa + Personas/Jornada + Wireframes + Protótipo testado + Redesign + Documentação (slide deck ou one-pager) + Pitch

💡 Dica Importante

As tarefas para casa são essenciais para consolidar o aprendizado. Cada aula tem uma tarefa curta e focada que prepara para a próxima aula. Não deixe acumular — o aprendizado é incremental!
