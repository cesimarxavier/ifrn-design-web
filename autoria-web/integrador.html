Roadmap de 21 Dias

Sua jornada semanal

Cada semana possui entregas paralelas. O time de Design prepara o terreno e o time de Dev constrói a estrutura.

Semana 1

Setup & Descoberta

Segunda • Dev

Ambiente & Estrutura

O código precisa existir e estar online, mesmo que vazio.

  • Repositório GitHub: Criado, público e com todos os membros adicionados.
  • Deploy Inicial: Link do Vercel/Netlify funcionando (Hello World).
  • Organização: Pastas /css, /img, /js criadas.
Terça • Design

Conceito Visual

Definição da "cara" do projeto antes de gastar tempo refinando.

  • Moodboard: Referências visuais reunidas.
  • Guia de Estilos v1: Paleta de cores (Hex) e Fontes definidas.
  • Wireframe: Rabisco estrutural da página (Papel ou Figma baixa fidelidade).

Semana 2

Layout & Alta Fidelidade

Segunda • Dev

Hard Coding (Desktop)

O site deve estar visualmente pronto em telas grandes.

  • HTML Semântico: Uso de header, main, section, footer.
  • CSS Grid/Flex: Layout estrutural funcionando (colunas, alinhamentos).
  • Estilização Base: Cores e fontes aplicadas via CSS Variables.
Terça • Design

Protótipo Final

O desenho pixel-perfect que guiará o código.

  • Alta Fidelidade (Desktop): Layout final no Figma com imagens reais.
  • Assets Exportados: Imagens cortadas e otimizadas (WebP/SVG).
  • Estados de Botão: Desenho de como os botões ficam no "Hover".

Semana 3

Responsividade & Interatividade

Segunda • Dev

Mobile First & JS

O site deve funcionar no celular e ter vida.

  • Media Queries: Site adaptado para Tablet e Celular (sem scroll horizontal).
  • JavaScript: Menu Hamburguer funcional e interações.
  • SEO Básico: Meta tags, Title e Favicon implementados.
Terça • Design

Adaptação & Apresentação

Como o site se comporta em telas pequenas?

  • Design Mobile: Tela do menu aberto desenhada no Figma.
  • Favicon: Ícone do navegador criado.
  • Slide de Defesa: Material visual para apresentação final.
Rubrica

Como você será avaliado

A nota máxima é 100 pontos. Avaliamos não apenas o resultado visual, mas a qualidade do código "por baixo do capô" e a funcionalidade da entrega.

Critério Eliminatório

Projetos não responsivos (que quebram no celular) ou sem deploy (link offline) terão penalidade máxima.

Design & UI

40 pts
  • ✓ Princípios CRAP: Contraste, Repetição, Alinhamento e Proximidade evidentes.
  • ✓ Tipografia e Cores: Leitura confortável e paleta harmônica (com contraste acessível).
  • ✓ Assets Otimizados: Imagens de alta qualidade visual, mas leves (WebP/SVG).
  • ✓ Fidelidade: O site reflete o protótipo aprovado na Semana 1.

Engenharia Web

40 pts
  • ✓ Semântica HTML5: Uso correto de tags estruturais (header, main, section).
  • ✓ Layout Moderno: Uso obrigatório de Flexbox e/ou CSS Grid.
  • ✓ Responsividade: Adaptação fluida para Mobile e Tablet (sem scroll horizontal).
  • ✓ Organização: Identação correta e estrutura de pastas limpa.

Funcionalidade & Processo

20 pts
  • ✓ JavaScript: Interações funcionais (Menu Mobile, Abas, Scroll).
  • ✓ Deploy: Site publicado e acessível (Vercel/Netlify).
  • ✓ Git/GitHub: Histórico de commits consistente durante as 3 semanas.
  • ✓ Performance: Carregamento rápido e imagens otimizadas.

Otimização de Imagens

Sites pesados são punidos pelo Google e irritam o usuário. Nunca suba imagens cruas da câmera ou do Unsplash direto para o código. Use estas ferramentas:

TinyPNG Comprime PNG e JPG sem perder qualidade visível. Squoosh (Google) Converte para WebP e redimensiona imagens. Favicon.io Gera o ícone que fica na aba do navegador.
