Fetch API no Alpine

Como trazer dados de fora para dentro do seu componente.

No Alpine, funções assíncronas funcionam nativamente. Você pode chamar uma API diretamente no x-data ou em métodos auxiliares.

O Padrão de Ouro

  • Estado Inicial: Defina arrays vazios e flags de loading.
  • Async/Await: Mais limpo que .then().
  • Try/Catch: Sempre proteja sua chamada de rede. 
<div x-data="{
    posts: [],
    async carregarPosts() {
        const resp = await fetch('https://api...');
        this.posts = await resp.json();
    }
}">

    <button @click="carregarPosts()">
        Baixar Dados
    </button>

    <template x-for="post in posts">
        <div x-text="post.title"></div>
    </template>

</div>

Lifecycle (Ciclo de Vida)

Quando carregar os dados automaticamente?

x-init

É o "construtor" do componente. Executa assim que o elemento é criado no DOM.

x-init="carregarDados()"

Exemplo Prático: Carregamento Automático

<!-- Assim que a div nasce, ela busca os dados -->
<div x-data="app()" x-init="init()">
    ...
</div>

<script>
function app() {
    return {
        usuarios: [],
        init() {
            this.usuarios = ['Ana', 'João'];
            console.log('Componente pronto!');
        }
    }
}
</script>

Laboratório: Crypto Dashboard

Construindo uma aplicação completa com API pública.

Resultado Final

CryptoTracker

Como construir?

1

Estrutura de Dados

Crie o x-data com um array vazio coins: [] e um booleano loading: true.

2

API Real

Use a API pública da CoinGecko ou simule com setTimeout (como no exemplo acima).

3

Skeleton UI

Use x-show="loading" para mostrar blocos cinzas pulsantes (animate-pulse do Tailwind) enquanto carrega.

<div x-init="fetchData()">
    
    <!-- Loading State -->
    <div x-show="loading">
        Carregando...
    </div>

    <!-- Data State -->
    <div x-show="!loading">
        <template x-for="coin in coins">
            <div x-text="coin.name"></div>
        </template>
    </div>

</div>
