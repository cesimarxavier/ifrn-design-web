1 Sistemas de Unidades CSS

O uso de pixels fixos é desencorajado em interfaces modernas. Adotamos unidades relativas para garantir acessibilidade e adaptação a múltiplos dispositivos.

PX (Absoluto)

Valor fixo. Não escala se o usuário alterar a fonte do navegador.

Uso: Bordas e sombras.
REM (Root EM)

Relativo à fonte raiz (html). Padrão de acessibilidade.

Uso: Fontes, margins, paddings.
% (Porcentagem)

Relativo ao tamanho do container pai.

Uso: Layouts fluidos e grids.
VH / VW

Relativo à janela de visualização (Viewport).

Uso: Hero Sections e Modais.
/* Exemplo de conversão de mentalidade */
html { font-size: 16px; } /* Base */

.card {
  width: 100%; /* Ocupa largura disponível do pai */
  padding: 1.5rem; /* 1.5 * 16 = 24px escalável */
  font-size: 1rem; /* 16px escalável */
  border: 1px solid #ddd; /* px ok para bordas */
}

Há diversas unidades de medidas que são úteis em diferentes contextos. Para saber mais sobre unidades de medidas, acesse o contéudo de unidades de medidas.

2 Propriedade Display e Backgrounds

Comportamento de Fluxo

display: block

Ocupa 100% da largura. Começa em nova linha. (Ex: div, section, p, h1).

display: inline

Ocupa apenas o tamanho do conteúdo. Não aceita width/height. (Ex: span, a, strong).

display: inline-block

Híbrido. Fica na mesma linha, mas aceita width/height/padding. Essencial para botões e cards simples.

Backgrounds Profissionais

O padrão de mercado para "Hero Sections" envolve o uso de background-size: cover e sobreposição de gradientes (overlay) para garantir leitura.

.hero {
  height: 80vh;
  background-image:
    linear-gradient(rgba(0,0,0,0.7), transparent),
    url('img.jpg');
  background-size: cover;
  background-position: center;
}

3 Propriedade Position

A propriedade position retira elementos do fluxo normal do documento. É a ferramenta principal para criar sobreposições, menus fixos e badges.

Relative

Mantém o espaço original, mas permite deslocamento.
Função principal: Servir de referência (âncora) para filhos absolutos.

Rel

Absolute

Flutua sobre tudo. Posiciona-se em relação ao pai relativo mais próximo. Ignora o fluxo (outros elementos sobem).

Abs (Top/Right)
Pai Relative

Fixed / Sticky

Fixed: Preso à janela (viewport). Não move com scroll.
Sticky: Relativo até atingir um ponto, depois fixa.

Prática em Sala: "Product Card"

Vamos consolidar os conceitos criando um componente de Card de Produto profissional. Este exercício combina: Display (layout), Position (badges e ícones) e Unidades (responsividade).

1

Estrutura e Display

Crie um container. Use inline-block para que possamos ter vários cards lado a lado futuramente.

.card { width: 300px; display: inline-block; background: white; border-radius: 8px; overflow: hidden; /* Corta a imagem nas bordas */ position: relative; /* Pai Âncora */ }
2

Imagem e Overlay

Insira uma imagem. Use um gradiente sobre ela se houver texto.

.card-img { width: 100%; height: 200px; object-fit: cover; /* Background-size para img tags */ }
3

Position Absolute (Badge)

Crie uma tag span para "Promoção" e posicione sobre a imagem.

.badge { position: absolute; top: 1rem; right: 1rem; background: #ef4444; color: white; padding: 0.25rem 0.5rem; font-size: 0.75rem; border-radius: 4px; }

Briefing Técnico

Você foi contratado para criar a página de espera ("Coming Soon") de uma startup de tecnologia. O design exige uma imagem de fundo imersiva e elementos fixos na interface.

1. Estrutura & Background

A página deve ocupar 100vh e não deve ter barra de rolagem. Use uma imagem de tecnologia como background e aplique um overlay escuro (rgba) para garantir leitura.

2. Formulário (Display)

O campo de input e o botão devem ficar na mesma linha. Utilize display: inline-block nos elementos ou no container.
Dica: Cuidado com o espaço em branco (whitespace) do inline-block.

3. Rodapé Fixo (Position)

A barra inferior com o copyright deve ficar presa à base da janela, mesmo se a tela for redimensionada. Use position: fixed ou absolute (se o pai for 100vh).

4. Centralização

Como ainda não estamos usando Flexbox oficialmente, tente centralizar o conteúdo principal usando margens automáticas e largura definida, ou text-align no container pai (body/main).
