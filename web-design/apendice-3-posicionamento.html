Especificação CSS Display Module Level 3

Guia Definitivo da Propriedade Display

Este documento cataloga todos os valores padronizados da propriedade display. Na especificação moderna, o display define dois papéis: o papel externo (como o elemento interage com os vizinhos) e o papel interno (como os filhos se comportam).

1. Valores de Fluxo (Outside)

Definem como a caixa do elemento é disposta no layout geral.

block

Gera uma caixa de bloco. Ocupa toda a largura e quebra linha antes e depois.

Ex: div, p, h1, section
inline

Gera uma caixa em linha. Flui com o texto. Ignora largura/altura e margens verticais.

Ex: span, a, strong
inline-block

Híbrido. Flui na linha (como inline), mas aceita propriedades de caixa (width, height, margin) como block.

flow-root IMPORTANTE

Gera uma caixa de bloco que estabelece um novo Contexto de Formatação de Bloco (BFC).

Uso Didático: É a forma moderna de fazer o "Clearfix". Ele contém elementos flutuantes (float) dentro dele sem colapsar a altura.

2. Sistemas de Layout (Inside)

Definem o contexto de formatação para os elementos filhos.

flex

Comporta-se como block externamente e ativa o Flexbox internamente (unidimensional).

inline-flex

Comporta-se como inline externamente e ativa o Flexbox internamente.

grid

Comporta-se como block externamente e ativa o Grid Layout internamente (bidimensional).

inline-grid

Comporta-se como inline externamente e ativa o Grid Layout internamente.

3. Modelo de Tabela (Legacy)

Permite que elementos não-tabela (divs, spans) se comportem como tabelas HTML. Muito usado em emails HTML ou layouts legados.

Valor Equivalente HTML Descrição
table <table> Define o container da tabela (bloco).
inline-table <table> Define uma tabela que flui inline com o texto.
table-row <tr> Agrupa células horizontalmente.
table-cell <td> Célula. Respeita vertical-align.
table-column <col> Define propriedades para colunas.
table-column-group <colgroup> Agrupa colunas.
table-header-group <thead> Grupo de cabeçalho (sempre no topo).
table-footer-group <tfoot> Grupo de rodapé (sempre no fundo).
table-row-group <tbody> Corpo da tabela.
table-caption <caption> Legenda da tabela.

4. Listas e Específicos

Valores para semântica específica.

list-item

Comporta-se como um <li>. Gera uma caixa principal de bloco e uma caixa de marcador (bullet/number) lateral.

Permite usar propriedades como list-style-type em divs ou spans.
ruby Nicho

Usado para anotações Ruby (comuns em tipografia do Leste Asiático).

Variações: ruby-base, ruby-text, ruby-base-container, ruby-text-container.

5. Visibilidade e Geração

none

Remove completamente o elemento da árvore de renderização. Não ocupa espaço, não responde a eventos e seus filhos também desaparecem.

contents

O elemento container "desaparece" para fins de estilização de layout, mas seus filhos permanecem visíveis.

Uso Prático: Fazer com que netos participem de um Grid Layout definido no avô, ignorando o pai (subgrid workaround).

Nota Didática: Sintaxe Multi-keyword

Na especificação mais recente (Level 3), a propriedade display suporta dois valores explícitos: um para o comportamento externo e outro para o interno.

display: flex;
// equivale a:
display: block flex;
display: inline-flex;
// equivale a:
display: inline flex;

Embora a sintaxe de uma palavra seja totalmente suportada e padrão, entender essa dualidade ajuda a explicar por que inline-flex existe.
